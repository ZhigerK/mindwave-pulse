**Продукт:** Платформа Mindwave (Гибридная образовательная экосистема)
**Версия:** 1.0 (Базовая логика)
**Статус:** Черновик

---

## 1. Введение и правила чтения

Этот документ описывает бизнес-правила системы в формате **«Событие (IF) ➔ Условие (AND) ➔ Действие (THEN)»**.
Все триггеры разделены по зонам ответственности и учитывают режим работы (Adult / Child).

**Обозначения целевых получателей:**
* **System:** Платформа выполняет действие фоном (без участия человека).
* **Mentor CRM:** Платформа создает карточку-задачу в рабочем пространстве Ментора.
* **Adult:** Уведомление уходит взрослому студенту (Email/Push/Telegram).
* **Parent:** Уведомление уходит родителю-наблюдателю.
* **Child:** Уведомление/анимация внутри интерфейса платформы.

## 6. Алгоритм Балансировки Нагрузки Менторов (Mentor Routing)

Чтобы система работала стабильно, распределение новых студентов (Триггер из п.2) работает по следующей логике (Round-Robin с весами):

1. **Проверка доступности:** Ментор должен быть в статусе `Active` (не в отпуске).
2. **Проверка компетенции:** Тег предмета Ментора совпадает с купленным курсом.
3. **Проверка Capacity (Емкости):** Количество текущих активных студентов у Ментора `< Max Limit` (например, 100).
4. **Назначение:** Студент отдается тому Ментору в пуле, у которого на данный момент меньше всего студентов в «Красной» и «Желтой» зонах (чтобы не перегружать тех, кто сейчас тушит пожары).
