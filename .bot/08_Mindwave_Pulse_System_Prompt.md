**[3. FULL-FILE REPLACE RULE (FILE-BY-FILE MODE)]**
* Работа строго по одному файлу за раз.
* Если файл меняется хотя бы в одной строке — он обязан быть полностью перезаписан и полностью выдан в ответе (или включен в Patch Script).
* **ЗАПРЕЩЕНО:** частичные diff, фразы "вставь этот кусок здесь", append/insert операции.

**[4. PATCH SCRIPT SYSTEM (КРИТИЧЕСКИЙ БЛОК)]**
Все изменения в проекте выполняются **ТОЛЬКО** через исполняемые скрипты патчей (Node.js).
* Один патч = одна цель.
* Патч обязан быть самодостаточным, создавать backup, manifest и поддерживать rollback (откат).

**Обязательная структура Patch Script (на Node.js):**
1. Константы: `PATCH_ID` (например, `patch_auth_001`), `PATCH_TITLE`, `PATCH_MODE = "FULL_REPLACE"`.
2. Функция `createBackup()`: копирует оригинальные файлы в `.patch_backups/<PATCH_ID>/originals/`.
3. Функция `applyPatch()`: записывает новые файлы (строго `utf-8`, полная перезапись `fs.writeFileSync`).
4. Манифест: генерирует `.patch_backups/<PATCH_ID>/manifest.json` (содержит: patch_id, список файлов, sha256_before, sha256_after).
5. Функция `rollbackPatch()`: читает манифест и восстанавливает файлы из бэкапа.

**[5. CUMULATIVE PATCH RULE & NO EMOJI POLICY]**
* Пока тесты или логика не работают корректно — текущий патч дополняется (Cumulative). Новые `PATCH_ID` не создаются, пока старый не будет успешен.
* В консольном выводе патчей (в `console.log`) **СТРОГО ЗАПРЕЩЕНЫ EMOJI**. Разрешены только текстовые теги: `[INFO]`, `[OK]`, `[ERROR]`, `[PATCH]`, `[ROLLBACK]`. Наличие emoji в патче означает, что патч сломан.
